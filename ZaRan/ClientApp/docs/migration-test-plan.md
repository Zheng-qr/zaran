# 功能迁移测试计划

## 测试概述

本文档描述了从 `my.vue` 到 `users/[id].vue` 功能迁移后的测试计划。

## 测试环境

- 开发服务器：http://localhost:3000
- 后端API：http://localhost:5127

## 测试用例

### 1. 路由跳转测试

#### 1.1 访问 /my 路由
- **测试步骤**：
  1. 在浏览器中访问 `http://localhost:3000/my`
  2. 观察页面行为
- **预期结果**：
  - 如果用户已登录：自动跳转到 `/users/[当前用户ID]`
  - 如果用户未登录：自动跳转到 `/login`
  - 显示加载动画和跳转提示

#### 1.2 直接访问用户主页
- **测试步骤**：
  1. 访问 `http://localhost:3000/users/[用户ID]`
  2. 检查页面内容
- **预期结果**：
  - 正确显示用户信息
  - 根据是否为自己的主页显示不同的功能

### 2. 功能完整性测试

#### 2.1 统计数量显示
- **测试项目**：
  - 作品数量统计
  - 收藏数量统计（仅自己可见）
  - 关注数量统计
  - 订单数量统计（仅自己可见）
- **测试方法**：
  1. 登录后访问自己的主页
  2. 检查统计数字是否正确显示
  3. 点击统计项是否能切换到对应标签页

#### 2.2 关注功能测试
- **测试步骤**：
  1. 访问他人的用户主页
  2. 点击"关注"按钮
  3. 检查按钮状态变化
  4. 刷新页面验证关注状态持久化
- **预期结果**：
  - 关注状态正确切换
  - 按钮文字和图标正确更新
  - 关注数量统计正确更新

#### 2.3 收藏查询测试
- **测试步骤**：
  1. 访问自己的主页
  2. 切换到"收藏"标签页
  3. 检查收藏列表显示
  4. 点击收藏项测试跳转
- **预期结果**：
  - 正确加载收藏列表
  - 收藏项显示正确的标题、类型、时间
  - 点击收藏项能正确跳转到对应页面

#### 2.4 标签页功能测试
- **测试项目**：
  - 作品标签页
  - 收藏标签页（仅自己可见）
  - 关注标签页
  - 订单标签页（仅自己可见）
  - 设置标签页（仅自己可见）
- **测试方法**：
  1. 逐一点击每个标签页
  2. 检查内容是否正确加载
  3. 验证权限控制是否正确
